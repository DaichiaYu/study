# study｜標籤系統 v0.1

本文件定義 `study` 倉庫在整理家庭教育、學習方法、親子關係、親密關係與 NTMY 故事素材時的標籤規則。

核心原則：

1. `study` 標籤是資料整理與案例檢索用，不等於 NTMY 的正式 `responseVector`。
2. NTMY 的 `forceVector` / `textureTags` 只在案例轉成 `ntmy_story_seed` 時使用。
3. 不為每一篇文章、每一種人物關係自由發明新標籤。
4. 情境、人物、年齡、關係角色要放在 metadata 或 scenario，不要污染底層機制標籤。

---

## 一、標籤分層

本倉庫標籤分三層：

```yaml
metadata:          # 來源與資料管理
classification:    # study 倉庫的教育／家庭問題分類
ntmy_mapping:      # 只有轉成 NTMY story seed 時才使用
```

---

## 二、metadata：來源與資料管理

metadata 用來回答：這份資料從哪裡來？是什麼形式？整理到什麼階段？

```yaml
metadata:
  source_teacher: 家衛 | 沈奕斐 | other
  source_course: 學習方法 | 自驅力 | 德育課 | 直播 | 鬆弛父母 | 家庭教育 | 社會學愛情思維 | 性教育課
  source_path: 原始檔案路徑
  content_type: source | ai_digest | case_card | family_playbook | ntmy_story_seed
  material_type: 主課 | 直播案例 | 問答 | 工具表 | PDF | 補充資料
  status: raw | drafted | reviewed | stable
```

### 欄位說明

- `source_teacher`：資料來源老師或系統。
- `source_course`：對應目前 README 裡的資料夾分類。
- `source_path`：原始逐字稿、PDF、xlsx 或 README 路徑。
- `content_type`：這份檔案在倉庫裡的加工階段。
- `material_type`：原始材料型態。
- `status`：整理狀態。

---

## 三、classification：study 倉庫主分類

classification 用來回答：這份資料在講什麼問題？背後機制是什麼？後續適合怎麼使用？

```yaml
classification:
  domain_tags: []
  problem_tags: []
  mechanism_tags: []
  child_need_tags: []
  adult_pattern_tags: []
  scenario_tags: []
  age_range: []
  suitable_for: []
```

---

## 四、domain_tags：問題領域

domain_tags 用來標記這份資料所屬的大領域。

```yaml
domain_tags:
  - learning_method        # 學習方法
  - self_drive             # 自驅力
  - moral_boundary         # 德育與邊界
  - parent_relaxation      # 父母鬆弛
  - family_system          # 家庭系統
  - elder_interference     # 長輩介入
  - school_relation        # 家校溝通
  - peer_relation          # 同儕關係
  - intimate_relation      # 親密關係
  - sex_education          # 性教育
```

使用規則：

- 每篇建議 1–3 個。
- 不要把具體情境當成 domain，例如 `孩子偷東西` 不放這裡。
- 若只是附帶提到，不必標。

---

## 五、problem_tags：表面問題

problem_tags 用來標記家長或當事人表面上會說出的問題。

```yaml
problem_tags:
  - homework_delay         # 作業拖延／做不完
  - attention_drift        # 注意力飄走／神遊
  - repeated_mistake       # 錯了又錯
  - low_motivation         # 缺乏動力
  - emotional_outburst     # 情緒爆發
  - rule_breaking          # 破壞規則
  - lying                  # 撒謊
  - stealing               # 偷東西
  - screen_overuse         # 手機／遊戲過度使用
  - parent_child_conflict  # 親子衝突
  - grandparent_spoiling   # 隔代溺愛／長輩破壞規則
  - school_communication   # 學校溝通問題
  - puberty_withdrawal     # 青春期退縮／鎖門／不溝通
  - relationship_conflict  # 親密關係衝突
  - sexual_boundary_issue  # 性邊界問題
```

使用規則：

- problem 是「表層症狀」，不等於原因。
- 一個案例可以有多個 problem，但不要超過 5 個。
- 若只是課程概念，不是具體問題，可留空。

---

## 六、mechanism_tags：背後機制

mechanism_tags 是最重要的 study 標籤，用來標記問題背後的教育／家庭／關係機制。

```yaml
mechanism_tags:
  - loss_of_control             # 掌控感喪失
  - toxic_pressure              # 毒性壓力
  - over_control                # 過度控制
  - over_protection             # 過度保護
  - over_correction             # 過度糾錯
  - boundary_collapse           # 邊界塌陷
  - rule_inconsistency          # 規則不一致
  - shame_driven                # 羞恥驅動
  - low_efficiency_training     # 低效率訓練
  - lack_of_scaffold            # 缺腳手架／缺台階
  - reward_dependency           # 外控獎勵依賴
  - emotional_misread           # 情緒誤判
  - adult_anxiety_projection    # 大人焦慮投射
  - intensive_parenting         # 密集母職／過度育兒
  - relationship_overextension  # 關係過度延伸／過度承擔
```

使用規則：

- 機制標籤回答「為什麼這件事會卡住」。
- 不要把人物角色寫進 mechanism，例如 `grandma_problem` 不應放這裡。
- 若尚不確定，放入 `candidate_tags`，不要直接新增正式標籤。

---

## 七、child_need_tags：孩子／當事人的真實需求

child_need_tags 用來標記表面行為背後可能需要被看見的需求。

```yaml
child_need_tags:
  - safety          # 安全感
  - autonomy        # 自主性
  - competence      # 能力感／勝任感
  - recognition     # 被看見／被肯定
  - belonging       # 歸屬感
  - boundary        # 邊界
  - structure       # 結構／流程／規則
  - play            # 遊戲與玩耍
  - exploration     # 探索
  - rest            # 休息
  - meaning         # 意義感
```

使用規則：

- 這組不只適用孩子，也可用於成人關係案例。
- 不要把需求直接道德化，例如「孩子就是想偷懶」不是需求標籤。
- 需要和 `mechanism_tags` 一起看。

---

## 八、adult_pattern_tags：大人常見誤區

adult_pattern_tags 用來標記「看似幫助，實則傷害」的大人反應模式。

```yaml
adult_pattern_tags:
  - helicopter_help              # 直升機式幫忙／包辦
  - corrective_reflex            # 反射性糾錯
  - comparison_pressure          # 比較壓力
  - emotional_blackmail          # 情緒勒索
  - inconsistent_rules           # 規則反覆
  - disguised_control            # 包裝成愛的控制
  - anxious_planning             # 焦慮式規劃
  - sacrifice_resentment         # 犧牲後怨懟
  - elder_good_intention_damage  # 長輩好心亂來
  - over_teaching                # 過度教導
```

使用規則：

- 這組專門標大人的行為模式，不標孩子。
- 若案例裡大人沒有明顯問題，可留空。
- 這組很適合後續做家庭 playbook。

---

## 九、scenario_tags：情境標籤

scenario_tags 用來描述具體場景，方便檢索，但不參與底層機制判斷。

```yaml
scenario_tags:
  - homework
  - exam
  - reading
  - math
  - chinese
  - english
  - chore
  - toy_cleanup
  - meal
  - sleep
  - school
  - teacher
  - grandparent
  - sibling
  - friend
  - romance
  - marriage
  - sexuality
```

使用規則：

- 情境標籤可以比較具體，但不要混進 mechanism。
- `grandparent` 是情境，不等於一定有 `elder_good_intention_damage`。
- `romance` 是情境，不等於一定有 `relationship_conflict`。

---

## 十、age_range：年齡階段

age_range 是弱欄位，只在明確相關時填。

```yaml
age_range:
  - infant
  - toddler
  - preschool
  - early_primary
  - upper_primary
  - puberty
  - adult
```

使用規則：

- 不強迫每篇都填。
- 只有關鍵期、青春期、小中高規劃、性教育等明顯與年齡相關的資料才填。
- 年齡不是此倉庫主索引，不要用年齡主導全部整理。

---

## 十一、suitable_for：後續用途

```yaml
suitable_for:
  - ai_digest
  - case_bank
  - family_playbook
  - ntmy_story_seed
  - essay_material
  - product_design
```

使用規則：

- 可複選。
- 若某篇只是理論框架，可能適合 `ai_digest` 與 `essay_material`，不一定適合 case。
- 若某篇有明確場景衝突，通常適合 `case_bank` 與 `ntmy_story_seed`。

---

## 十二、NTMY 轉譯層

NTMY 轉譯層只在案例要轉成 NTMY 故事素材時使用。

NTMY 正式語法：

```text
forceVector = 反應往哪裡動
textureTags = 反應用什麼材料／質地在動
```

study 裡的 `problem_tags`、`mechanism_tags`、`scenario_tags` 不得直接拿來當 NTMY 的 forceVector 或 textureTags。

```yaml
ntmy_mapping:
  usable_for_ntmy: true | false
  scenario_type: family | school | peer | romance | self_growth | work | survival
  tension_axis: []
  source_forceVector: []
  source_textureTags: []
  target_forceVector_candidates: []
  target_textureTags_candidates: []
  movement_note:
```

### 可用 tension_axis 候選

`tension_axis` 是 story seed 用，不是 NTMY 正式 responseVector。

```yaml
tension_axis:
  - safety_vs_autonomy        # 安全感 vs 自主性
  - love_vs_control           # 愛 vs 控制
  - protection_vs_growth      # 保護 vs 成長
  - rule_vs_flexibility       # 規則 vs 彈性
  - obedience_vs_judgment     # 服從 vs 判斷
  - honesty_vs_self_protect   # 誠實 vs 自保
  - relation_vs_boundary      # 關係 vs 邊界
  - effort_vs_meaning         # 努力 vs 意義
  - desire_vs_responsibility  # 慾望 vs 責任
```

### NTMY forceVector 候選

正式值以 NTMY 文件為準，目前只可使用：

```yaml
source_forceVector:
  - attach
  - cut
  - hold
  - push
  - freeze
  - mask
  - scatter
  - clarify
```

### NTMY textureTags 候選

正式值以 NTMY 文件為準，目前只可使用：

```yaml
source_textureTags:
  - emotional
  - rational
  - relational
  - principled
  - verbal
  - somatic
  - positional
  - protective
  - survival
```

---

## 十三、candidate_tags：候選標籤管理

如果遇到現有標籤無法表達的新概念，先放候選，不直接升正式。

```yaml
candidate_tags:
  - name:
    layer: domain | problem | mechanism | child_need | adult_pattern | scenario | tension_axis
    reason:
    seen_in:
    merge_candidate:
```

新增正式標籤需符合以下條件之一：

1. 既有標籤無法表達此教育／家庭／關係機制。
2. 此標籤會影響後續 playbook 建議或案例判斷。
3. 此標籤至少預期服務 3 篇以上資料或案例。
4. 雖使用量少，但對 NTMY story seed 或家庭決策非常關鍵。

---

## 十四、禁止事項

- 不得把人物角色直接寫成 mechanism，例如 `grandma_problem`。
- 不得把單一情境寫成 domain，例如 `toy_cleanup_domain`。
- 不得把 study 的 problem_tags 直接當 NTMY forceVector。
- 不得為每一篇逐字稿新增一次性標籤。
- 不得只因語氣不同就新增標籤。
- 不得把年齡階段當作唯一主索引。
- 不得把 NTMY 的 `forceVector` / `textureTags` 顯示成玩家可見文字。
- 不得把 NTMY movement 當作道德分數。

---

## 十五、範例

### 範例一：長輩包辦孩子

```yaml
metadata:
  source_teacher: 沈奕斐
  source_course: 家庭教育
  content_type: case_card
  material_type: 主課

classification:
  domain_tags: [family_system, elder_interference]
  problem_tags: [grandparent_spoiling]
  mechanism_tags: [over_protection, loss_of_control, rule_inconsistency]
  child_need_tags: [autonomy, competence, boundary]
  adult_pattern_tags: [helicopter_help, elder_good_intention_damage]
  scenario_tags: [grandparent, chore]
  age_range: [early_primary]
  suitable_for: [case_bank, family_playbook, ntmy_story_seed]

ntmy_mapping:
  usable_for_ntmy: true
  scenario_type: family
  tension_axis: [safety_vs_autonomy, love_vs_control, protection_vs_growth]
  source_forceVector: [push, attach]
  source_textureTags: [protective, relational]
```

### 範例二：作業做不完

```yaml
metadata:
  source_teacher: 家衛
  source_course: 直播
  content_type: case_card
  material_type: 直播案例

classification:
  domain_tags: [learning_method, self_drive]
  problem_tags: [homework_delay, attention_drift]
  mechanism_tags: [lack_of_scaffold, toxic_pressure, over_correction]
  child_need_tags: [structure, competence, autonomy]
  adult_pattern_tags: [corrective_reflex, anxious_planning]
  scenario_tags: [homework, school]
  age_range: [early_primary, upper_primary]
  suitable_for: [case_bank, family_playbook, ntmy_story_seed]

ntmy_mapping:
  usable_for_ntmy: true
  scenario_type: school
  tension_axis: [effort_vs_meaning, love_vs_control]
  source_forceVector: [freeze]
  source_textureTags: [emotional, rational]
```

### 範例三：密集母職／父母過度投入

```yaml
metadata:
  source_teacher: 沈奕斐
  source_course: 鬆弛父母
  content_type: ai_digest
  material_type: 主課

classification:
  domain_tags: [parent_relaxation, family_system]
  problem_tags: [parent_child_conflict]
  mechanism_tags: [intensive_parenting, adult_anxiety_projection, boundary_collapse]
  child_need_tags: [autonomy, boundary, safety]
  adult_pattern_tags: [anxious_planning, sacrifice_resentment, disguised_control]
  scenario_tags: [homework, school]
  suitable_for: [ai_digest, essay_material, family_playbook]

ntmy_mapping:
  usable_for_ntmy: true
  scenario_type: family
  tension_axis: [love_vs_control, relation_vs_boundary]
  source_forceVector: [attach, push]
  source_textureTags: [protective, relational]
```
